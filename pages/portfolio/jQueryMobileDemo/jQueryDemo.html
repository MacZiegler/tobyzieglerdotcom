This was originally a lorem ipsum placeholder text, but I thought it would be more useful if it described the project. If you would rather just see the lorem ipsum, it is just a click away.

This page was an assignment for Advanced Web Development as an example jQuery demonstration. The criteria were as follows:

Build a two-page jQuery Mobile site. If possible, try testing the site from a mobile device or emulator.

  1. Add a page for "Home" and a page for "Contact".
  2. For each page, add an appropriate header, content area, and footer, with some sample content.
  3. Add a navbar for each header, with appropriate links to each of the two pages; add classes to ensure the appropriate nav item shows as active on each page.
  4. Add a paragraph of text at the bottom of each page's content area.
  5. At the bottom of the page, add an event handler: set the text of each .orientation paragraph to the appropriate value ("portrait" or "landscape"); note that the orientationchange event is called on page load, to set the text when the page first loads.
  6. Visit the jQuery Mobile website to find one other UI element or event to add to your site.

For the final requirement (number six), I made the lorem ipsum text and this extended explanation text swap out with a click. I also created a couple of buttons on the bottom to navigate back to the home page of the hosting site, or just back to the portfolio section.

One final bonus: I included the screen example from the assignment as a popup graphic.

So, you would like to know more? If not, it's easy to go back.

How about this tidbit: I lost many hours of work over the course of about a week trying to get the styles to just plain work. I ripped the code apart and rebuilt it. I checked for typos, letter by letter. I pulled out hair. I did something else and came back again. I read some more. I studied some more. I found examples. I tried again.

Nothing styled. Just plain html, no matter what. Even when I eventually directly copied an example into a blank page: Nothing.

That did give me an idea, though. I was using the fun and cool Cloud9 IDE. It is what we started with in the advanced web development class this page was coded for. Maybe the problem might be the IDE and not me? Surely not.

That was it. When I moved the page to a local drive and viewed it through my local servers, the buttons and divisions and backgrounds just popped right into place.

Sheesh.

I guess I will just chock this one up to "lesson learned".
